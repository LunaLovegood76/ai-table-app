## 一、现状盘点

### 1.1 当前 Savy 素材库（共 14 个 SVG）

| 类别 | 数量 | 文件列表 |
|------|:----:|---------|
| **课程表情** | 4 | `savy-celebrate` · `savy-happy` · `savy-sad` · `savy-thinking` |
| **场景插画** | 10 | `savy-love` · `savy-angry` · `savy-gaze` · `savy-coffee` · `savy-chart` · `savy-reading` · `savy-rocket` · `savy-trophy` · `savy-wink` · `savy-yawn` |

### 1.2 设计规范（已有）

- **角色**：玻尿酸鸭（Lalafanfan Duck）
- **特征**：黄色圆头、粉色腮红、橙色鸭嘴、金色圆框眼镜、粉色项圈+金色铃铛
- **表情差异手段**：眼睛形态 / 嘴巴形态 / 手臂动作 / 装饰元素
- **文件规范**：`savy-{emotion}.svg`，尺寸 `120×120`

### 1.3 核心问题

| 问题 | 说明 |
|------|------|
| 🗣️ **文案无"鸭"味** | 所有反馈文案都是通用的"太棒了！""再试一次"，完全没有利用鸭子 IP 的谐音梗 |
| 😐 **情绪颗粒度不足** | 课程中实际使用的只有 4 种表情，场景插画 10 个但未在课程流程中使用 |
| 🖼️ **全部静态** | 14 个 SVG 全是静态图，缺少动态表现力 |
| 📦 **表情与激励脱钩** | 表情只是装饰，没有和游戏化体系（徽章/等级/打卡）产生联动 |
| 😇 **太"正经"** | 缺少反差萌、沙雕感，情感共鸣不够强 |

## 二、ChineseBQB 关键启发

### 2.1 开心鸭系列（最直接的竞品参考）

ChineseBQB 收录了两个开心鸭系列（008 + 057），合计 **58 张**表情。其核心设计策略：

#### 启发 ①：「鸭」字谐音梗文案体系

开心鸭系列最大的杀手锏是**把所有"呀/啊"替换成"鸭"**，形成了一套完整的谐音梗文案：

#### 启发 ②：场景化 > 纯情绪

高传播表情不只是表达情绪，而是**绑定了具体生活场景**：
- ❌ 纯情绪："开心"（传播力弱）
- ✅ 场景+情绪："刚下班鸭"、"被生活掐住了肚皮"、"气到发胖"（传播力强）

#### 启发 ③：反差萌是最强传播密码

可爱外表 + 反差台词 = 病毒式传播：
- "你说我是不是你的小可爱"（可爱）
- "我这一鸭翅下去你可能会死"（反差）
- "可爱真的是可以为所欲为"（自嘲式反差）

### 2.2 沙雕企鹅系列（动态表现力标杆）

157 张表情**全部是 GIF 动态图**，是仓库中动物 IP 收录量最大的系列。

**启发**：动态 > 静态。同样的情绪，加上动画后传播力翻倍。

### 2.3 小幺鸡系列（双版本策略）

129 张表情中，每个情绪都有 **GIF 动态版 + JPG 静态版** 两个版本（文件名成对出现）。

**启发**：可以为核心表情同时提供静态 SVG + CSS 动画版，在不同场景使用不同版本。

### 2.4 整体仓库组织（收集性启发）

- 每个系列有编号（001-107）
- 每张表情有编号（00001-00050）
- 用户可以下载整个系列的 zip 包
- 在线搜图斗图工具

**启发**：表情可以变成"可收集"的资产，和游戏化体系联动。

## 三、增强方案

### 3.1 🗣️ P0 —「鸭」字谐音梗文案体系

> **实施难度**：⭐ 极低（纯文案替换）
> **预期效果**：⭐⭐⭐⭐⭐（立竿见影提升 IP 辨识度和情感共鸣）

#### 3.1.1 设计原则

1. **所有"呀/啊"结尾的语气词 → 替换为"鸭"**
2. **"压力" → "鸭力"**，**"优雅" → "优鸭"** 等自然谐音
3. **不强行谐音**，只在自然、好笑的地方用，避免尬
4. **保留原有信息传达**，谐音梗是锦上添花不是喧宾夺主

### 3.3 🎬 P1 — 核心表情 CSS 动画

> **实施难度**：⭐⭐ 较低（纯 CSS，不需要新资源）
> **预期效果**：⭐⭐⭐⭐（动态感大幅提升）

#### 3.3.1 动画方案

不需要制作 GIF，利用 CSS `@keyframes` 为 SVG 容器添加动画：

| 表情 | 动画名 | 动画效果 | CSS 关键帧 |
|------|--------|---------|-----------|
| `celebrate` | `savy-bounce-celebrate` | 弹跳 + 旋转 + 放大缩小 | `0%{transform:scale(0.8) translateY(20px)} 50%{transform:scale(1.1) translateY(-10px) rotate(5deg)} 100%{transform:scale(1)}` |
| `happy` | `savy-wave` | 左右轻微摇摆（像挥手） | `0%{transform:rotate(-3deg)} 50%{transform:rotate(3deg)} 100%{transform:rotate(-3deg)}` |
| `sad` | `savy-shake-sad` | 轻微颤抖 + 下沉 | `0%{transform:translateX(-2px)} 25%{transform:translateX(2px)} 50%{transform:translateX(-2px) translateY(3px)} 100%{transform:translateX(0)}` |
| `thinking` | `savy-float` | 缓慢上下浮动（像在思考） | `0%{transform:translateY(0)} 50%{transform:translateY(-5px)} 100%{transform:translateY(0)}` |
| `cool` | `savy-slide-in` | 从右侧滑入 + 微微倾斜 | `0%{transform:translateX(30px) rotate(10deg); opacity:0} 100%{transform:translateX(0) rotate(0); opacity:1}` |
| `surprised` | `savy-pop` | 突然弹出 + 放大 + 回弹 | `0%{transform:scale(0)} 60%{transform:scale(1.3)} 100%{transform:scale(1)}` |
| `cheer` | `savy-pump` | 向上冲拳 | `0%{transform:translateY(0)} 30%{transform:translateY(-15px) scale(1.1)} 60%{transform:translateY(-5px)} 100%{transform:translateY(0)}` |
| `trophy` | `savy-glow` | 金色光晕脉冲 | `0%{filter:drop-shadow(0 0 0 gold)} 50%{filter:drop-shadow(0 0 12px gold)} 100%{filter:drop-shadow(0 0 0 gold)}` |

#### 3.3.2 实现方式

```css
/* 答对时自动播放 */
.savy-emotion[data-emotion="celebrate"] {
  animation: savy-bounce-celebrate 0.6s ease-out;
}

.savy-emotion[data-emotion="sad"] {
  animation: savy-shake-sad 0.4s ease-in-out 2; /* 颤抖 2 次 */
}

.savy-emotion[data-emotion="thinking"] {
  animation: savy-float 2s ease-in-out infinite; /* 持续浮动 */
}

/* 里程碑弹窗中的 trophy 持续发光 */

#### 3.5.2 彩蛋触发逻辑

```
触发条件 → 彩蛋表情替换常规表情 → 用户看到意外的反差萌 → 惊喜感 → 想截图分享

- **低频出现**：不是每次都触发，保持稀缺感

### 3.6 🔓 P2 — 表情解锁收集系统

> **实施难度**：⭐⭐⭐⭐ 较高（需要新增状态管理 + UI）
> **预期效果**：⭐⭐⭐⭐⭐（长期留存核心机制）

#### 3.6.1 设计理念

将 Savy 表情从"装饰品"变成"可收集的资产"，和游戏化体系深度联动：
