## 一、多邻国游戏化激励机制全景图

多邻国的游戏化设计被公认为行业标杆，其激励体系可拆解为 **8 个核心维度**：

| # | 维度 | 核心设计思路 |
|:-:|------|-------------|
| ① | XP 经验值 | 每课固定 XP + 完美加成 + 限时双倍 XP + XP 加速器 |
| ② | 连续打卡（Streak） | 🔥 火焰天数 + 断签推送 + Streak Freeze + 里程碑庆祝 + 好友对比 |
| ③ | 成就徽章 | 多维度（课程/XP/连续/速度/社交）+ 分等级（铜/银/金）+ 进度条 + 解锁仪式 |
| ④ | 排行榜（联赛） | 周赛制 + 联赛分级（青铜→黑曜石）+ 前10晋级/后5降级 + 好友排行 |
| ⑤ | 段位/等级 | 等级递进 + 升级进度条 + 升级庆祝动画 + 差异化图标 |
| ⑥ | 即时反馈 | 答对绿色+音效 / 答错红色+解释 / 完成撒花 / 音效 / 触觉反馈 |
| ⑦ | 生命值/红心 | 5 颗红心 + 答错扣心 + 等待恢复或付费 + 练习旧课恢复 |
| ⑧ | 虚拟货币/商店 | 宝石货币 + 商店（Streak Freeze/服装）+ 宝箱 + 吉祥物换装 |

## 三、增强方案（本轮实施）

本轮优先实施 3 项高投入产出比的增强：

### 3.1 🔥 功能 1：连续打卡情感化（断签弹窗 + 里程碑庆祝）

> 不做 Streak Freeze 道具

#### 3.1.1 断签弹窗

**触发时机**：`updateStreak()` 检测到 `diffDays > 1` 时（即断签了）

> **⚠️ 关于"连续学习天数"的定义与检测时机调整**
>
> 当前 demo 没有独立的"打卡"按钮，采用的是**多邻国同款逻辑**：**完成一节课程 = 当天打卡成功**。`updateStreak()` 会对比 `lastStudyDate`（上次学习日期）和今天的日期来计算连续天数。
>
> **现有问题**：断签检测只在 `completeLesson()` 时触发，用户断签后打开 app 但没完成课，不会看到断签弹窗。
>
> **本次调整**：将断签/里程碑检测从 `completeLesson()` **提前到 app 初始化时**（`render()` 首次调用），这样用户一打开页面就能看到断签弹窗或里程碑庆祝。`completeLesson()` 中仍保留 `updateStreak()` 来处理"今天首次完成课程"的连续天数更新。

**弹窗内容**：

| 元素 | 内容 |
|------|------|
| 吉祥物 | `savy-sad.svg`（泪汪汪大眼 + 委屈嘟嘴） |
| 标题 | "哎呀，连续记录断了…" |
| 副标题 | "你之前已经连续学习了 **{N} 天**！" |
| 鼓励文案 | "没关系，重新开始也很棒！今天学一课就能开启新的连续记录 🔥" |
| 按钮 | "重新出发！"（绿色主按钮） |
| 背景 | 半透明遮罩（复用 `locked-tooltip-overlay` 样式） |
| 动画 | `popIn` 弹入动画（已有） |
| 音效 | `playStreakBreak()` — 柔和下降音 |

**新增状态字段**：
- `previousStreak`：记录断签前的连续天数（用于弹窗展示"你之前连续了 N 天"）
- `streakBrokenShown`：标记本次断签弹窗是否已展示过（避免重复弹出）

### 3.3 🎵 功能 3：音效系统

#### 3.3.1 技术方案

使用 **Web Audio API 实时合成**，零资源文件依赖：
- 不需要额外下载音频资源
- 体积为零，纯代码生成
- 风格统一，都是简洁的电子音
- 和多邻国的简短提示音风格一致

试听页面：`demo/preview-sounds.html`

#### 3.3.2 音效清单

| 场景 | 方法名 | 描述 | 波形 | 音符序列 | 时长 |
|------|--------|------|------|---------|------|
| 答对 | `playCorrect()` | 清脆上升双音 | sine | C5(523Hz) → E5(659Hz) | ~0.3s |
| 答错 | `playIncorrect()` | 低沉下降双音 | sine | E4(330Hz) → C4(262Hz) | ~0.3s |
| 完成课程 | `playComplete()` | 欢快三连音 | sine | C5 → E5 → G5(784Hz) | ~0.8s |
| 获得徽章 | `playBadge()` | 闪亮颤音 | triangle | E6(1319Hz) 颤音 | ~0.6s |
| 升级段位 | `playLevelUp()` | 号角四连音 | square | C4→E4→G4→C5 | ~1.0s |
| 连续里程碑 | `playMilestone()` | 温暖上升+闪亮尾音 | sine+triangle | G4→B4→D5 + E6 | ~1.0s |
| 断签 | `playStreakBreak()` | 柔和下降音 | sine | G4(392Hz) → D4(294Hz) | ~0.5s |

#### 3.3.3 SoundManager 设计

```javascript
const SoundManager = {
  enabled: true,       // 用户可关闭
  audioContext: null,   // 延迟初始化（需要用户交互后才能创建）

  init()               // 首次用户交互时初始化 AudioContext
  playCorrect()        // 答对
  playIncorrect()      // 答错
  playComplete()       // 完成课程
  playBadge()          // 获得徽章
  playLevelUp()        // 升级段位
  playMilestone()      // 连续里程碑
  playStreakBreak()     // 断签

  toggle()             // 开关音效
}
```
